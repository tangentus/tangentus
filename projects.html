WebScrape API

This is still in the works in terms of direction and vision. My goal for this project is to provide a way to trigger customizable web-scrapes via API, or by "building" a scrape via a user-interface. Currently, the majority of my work has gone into the API portion.

The "cools" and "neats"

  • 😎 MongoDB because of its malleability. Scrapes are anything but well-defined and having a schema that can change with ease is quite a benefit.
  • 😎 Designed to be able to add scrape strategies for grabbing links, images, etc.
  • 🧐 Usage of Ruby's >> operator for Proc objects. It passes the result of the first Proc to the second Proc Used here!
  • 🧐 A pattern for fetching and parsing websites in a way that allows for composition. The fetchable and parsable Concerns give access to their logic via a Proc. This allows it to be passed around and applied with ease, and makes it easy to use function composition with the >> operator aforementioned. Check it out here!
  • 🧐 Resque as the ActiveJob background job processor. As it stands, it is fairly dumb. But being able to offload larger scrapes was important from the get-go.
